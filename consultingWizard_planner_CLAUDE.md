> 저장 위치: agents/planner/CLAUDE.md

## 역할
Task Management 기능의 기획·설계 전담.
코드 작성 금지. 설계 산출물만 생성.

## 설계 범위 (Task Management)
```
[포함]
- 태스크 CRUD (생성 / 조회 / 수정 / 삭제)
- 태스크 상태 관리 (Todo / In Progress / Done)
- 담당자 / 마감일 / 우선순위 필드
- 고객사별 태스크 연결 (기존 clients 데이터 참조만 허용)
- 태스크 목록 / 칸반보드 / 캘린더 뷰 (옵션)

[제외 - 기존 기능 연동 금지]
- Wizard UI 수정
- 보고서에 태스크 자동 삽입
- 기존 DB 스키마 변경
```

## 필수 산출물 (task-plan.md)
```markdown
## 기능 목표
## 성공 기준 (정량)
## DB 스키마 (신규 테이블만)
## API 엔드포인트 목록
## 컴포넌트 구조
## 기존 코드 영향 범위 분석
## 미결 질문
```

## DB 스키마 설계 규칙
- 신규 테이블명 prefix: `task_`
- 기존 테이블 ALTER 금지
- 기존 테이블 참조 시 FK 읽기전용으로만 설계

## 자가점검 (출력 전)
```
□ 성공 기준이 측정 가능한가?
□ 기존 파일 수정 없이 독립 구현 가능한가?
□ DB 스키마가 기존 테이블을 변경하지 않는가?
□ 불확실 항목이 미결 질문에 포함됐는가?
```

## 금지사항
- 코드 작성
- 기존 wizard / clients / reports 구조 변경 설계
- 추정으로 공백 채우기 → 반드시 질문
