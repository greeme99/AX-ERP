# AX ERP — UI Identity 가이드 (v1)

`/Users/greeme/Pictures/이미지_참조` 16개 레퍼런스(Boltshift, KUB74, EduNova AI ERP, FlowAI, Vault,
Smile360, 국내 회계 ERP, 보험 청구관리 등)에서 공통으로 반복되는 패턴을 추출해 정의한 AX-ERP
전용 디자인 아이덴티티다. **정보 항목과 기능은 그대로 두고 시각 스타일·레이아웃만 통일**하는 것이
목표 — 신규 화면/기능 추가가 아니다.

## 1. 왜 이 방향인가 (레퍼런스 공통점)

16개 중 12개가 "흰색/연회색 캔버스 + 블루 액센트 + 좌측 사이드바 + 카드형 KPI + 컬러 상태뱃지"
조합을 쓴다(Boltshift, KUB74, Vault, Smile360, 국내 회계ERP, EduNova 등). 이 조합이 엔터프라이즈
ERP에 가장 잘 맞는 이유: 파란색은 금융/신뢰 톤에 적합하고, 우리 앱이 이미 로그인/역할(RBAC) 기반
이라 사이드바+역할뱃지 패턴과 궁합이 좋다. 나머지 4개(FlowAI 모노톤, 보험청구 필터그리드, 거래내역
확장형 로우, 역할별 컬러 대시보드)에서는 컴포넌트 레벨 아이디어만 차용한다: 상태 뱃지 색상표기,
필터바 그리드, 펼쳐지는 상세 로우, 역할별 뱃지 컬러.

관리자=`#7C3AED`(보라), 영업담당=`#2563EB`(블루), 구매담당=`#0891B2`(틸), 생산담당=`#EA580C`(오렌지).

## 3. 타이포 / 여백 / 형태

- 폰트: 시스템 기본(Pretendard/Apple SD Gothic/Segoe UI 순 폴백), 숫자는 tabular-nums.
- 크기: 페이지 타이틀 20px(bold) · 섹션 타이틀 15px(semibold) · 본문/표 13px · 캡션 11–12px(secondary).
- 카드 radius 12px, 버튼/인풋 radius 8px, 배지(pill) radius 999px.
- 그림자: `0 1px 2px rgba(16,24,40,.04), 0 4px 10px rgba(16,24,40,.04)` — 과하지 않게.
- 카드 패딩 20–24px, 카드 간 간격 16px, 표 셀 패딩 10–12px.

## 4. 레이아웃 셸

- **사이드바(좌, 224px 고정)**: 상단 로고, 메뉴 그룹(대시보드 / 기준정보 / 영업·구매·생산·재고 /
  회계·승인함 / 연동로그·AI Agent·참고데이터), 항목은 아이콘+라벨, 활성 항목은 `--brand-soft` 배경
  + `--brand` 좌측 4px 바. 역할에 없는 메뉴는 기존처럼 아예 숨김(hasRole 로직 유지).
- **상단바**: 페이지 타이틀(좌), 검색(선택), 알림 아이콘(연동 이벤트/승인 대기 뱃지), 사용자
  아바타+이름+역할뱃지+로그아웃(우).
- **본문**: 캔버스 배경 위에 카드 단위로 섹션 분리. 목록 페이지는 `필터바 카드` → `테이블 카드`
  순서 고정.

## 5. 공통 컴포넌트 규칙

- **KPI 카드**: 캡션(라벨) → 큰 숫자 → 보조 텍스트/추세. 대시보드 최상단 4개 고정.
- **상태 뱃지**: 배경 연한 색 + 글자 진한 색, 위 컬러 토큰의 semantic 매핑을 전 화면에서 동일하게
  적용(예: `ACTIVE`/`완료`/`정상`은 항상 success, `PENDING`/`대기`는 항상 warning).
- **테이블**: 헤더 행 `--bg-canvas` 배경+secondary 텍스트, 짝수 행 살짝 음영 또는 hover 시 `--brand-soft`,
  숫자 컬럼 우측 정렬, 액션은 행 끝 아이콘 버튼(텍스트 버튼 지양).
  긴 텍스트(연동로그 payload 등)는 뱃지/축약 처리.
- **필터바**: 라벨+인풋 그리드(2~4열), 우측 검색 버튼은 `--brand` 채움, 보조 액션(새로고침/내보내기)은
  아웃라인 버튼.
- **AI Agent 전용**: 추천/인사이트 카드에 좌측 얇은 그라디언트(`--brand`→보라) 바 + "AI 설명" 텍스트는
  살짝 톤다운된 배경 박스로 구분해, 규칙기반 수치와 AI 서술을 시각적으로 분리.
- **빈 상태**: "추천 없음"/"인사이트 없음" 등은 아이콘+회색 텍스트 1줄로 통일(현재 텍스트만 있는 것과
  동일 문구, 스타일만 통일).

## 6. 페이지별 적용 메모 (정보항목은 기존 그대로, 배치/스타일만 변경)

| 페이지 | 기존 정보항목 (변경 없음) | 레이아웃 보완 포인트 |
|---|---|---|
| 로그인 | 이메일/비밀번호, 데모계정 안내 | 중앙 카드 + 브랜드 컬러 좌측 비주얼 패널(또는 상단 로고), 에러 메시지는 danger 톤 인라인 |
| 대시보드 | KPI(dashboard/kpi) | 4카드 고정 그리드 + 트렌드 표시, 하단에 모듈 바로가기 타일 추가 없이 기존 수치만 카드화 |
| 기준정보(MDM) | 품목/고객/공급사/창고/공장 목록+등록폼 | 좌: 등록 폼 카드(관리자만), 우: 목록 테이블 카드로 2열 분리 |
| 영업 | 수주 목록/등록, 출하 | 필터바 없음(단순 목록) → 상태뱃지(DELIVERED 등) 적용, 출하 버튼은 행 액션 아이콘화 |
| 구매 | PR/PO 목록/등록, 입고처리 | 승인 상태 뱃지 통일, PR→PO 연결 정보는 보조텍스트로 |
| 생산 | 생산오더/작업지시/실적입력, 품질검사 등록 | 진행 상태(PLANNED/IN_PROGRESS/COMPLETED) 뱃지, 시리얼 생성 체크박스는 폼 카드 내 강조 |
| 재고 | inventory, LOT/시리얼, 정합성 점검 | 정합성 "부정합" 행은 danger 톤 하이라이트(보험청구 화면의 위험행 강조 차용), LOT 추적은 펼침형 상세(거래내역 화면 패턴 차용) |
| 회계 | 전표 목록 (관리자 전용) | 관리자 전용 안내 배지, 차변/대변 표는 국내 회계ERP 참고 톤 유지 |
| 승인함 | approval_workflow | PENDING/APPROVED/REJECTED 뱃지, 승인/반려 버튼은 아이콘+색상(초록/빨강) |
| 연동 로그 | integration_event_log | source_system(MES/WMS) 아이콘 구분, status 뱃지, payload는 "보기" 토글로 축약 |
| AI Agent | 5개 에이전트 추천/인사이트 + ai_narrative | 에이전트별 카드 헤더(아이콘+이름), rationale는 표, ai_narrative는 강조 박스 |
| 참고 데이터 | 수요예측/품질검사/출하/재무월요약/KPI/AI로그 | 탭 형태로 6개 서브테이블 전환(현재 세로나열 → 탭으로 화면 스크롤 축소) |

## 7. 비범위
