# Task Plan — JWT 토큰 포맷 전환

## 설계 결정 (사용자 확인)
- **세션 테이블은 유지, 토큰 포맷만 표준 JWT로 전환.** 완전 무상태(stateless) JWT는 로그아웃 즉시
  무효화가 불가능해지는 트레이드오프가 있어 이번 범위에서는 채택하지 않음.
- 신규 의존성 없음 — `hmac`/`base64`/`json`/`hashlib` 표준 라이브러리만으로 HS256 서명을 직접 구현
  (`app/auth.py`의 `_jwt_encode`/`_jwt_verify`).
- 검증 흐름: ① JWT 서명 검증(위조 토큰을 DB 조회 없이 즉시 401) → ② 세션 테이블 대조(로그아웃/만료는
  여전히 DB가 기준). 서명이 유효해도 세션 테이블에 없거나 만료됐으면 401.
- `JWT_SECRET` 환경변수로 서명 키 교체 가능(기본값은 개발용 고정 문자열 — 운영 전환 시 필수 교체).
- payload 클레임: `sub`(user_id), `iat`, `exp`(세션 테이블 `expires_at`과 동일 시각에서 계산해
  두 값이 어긋나지 않게 함), `jti`(토큰 유일성 보장, 세션 테이블 PK 충돌 방지).

## 비범위
- 완전 무상태 전환(세션 테이블 제거) — 로그아웃 즉시무효화를 포기해야 해서 채택 안 함
- 리프레시 토큰, 토큰 재발급 없이 만료 연장 등 — 이번 범위 밖

## 성공 기준
1. 발급된 토큰이 `header.payload.signature` 3파트 구조(표준 JWT 형식)
2. 헤더에 `alg: HS256`, payload에 `sub`/`iat`/`exp` 포함
3. 서명이 위조된 토큰은 DB 조회 전에 401로 즉시 차단
4. payload를 조작한 토큰(역할 상승 시도 등)도 서명 불일치로 401
5. 정상 토큰은 기존과 동일하게 로그인/조회/로그아웃 전부 정상 동작(회귀 없음)
