# Task Plan — 시리얼 UI 고도화

## 설계 결정 (사용자 확인)
- **출하 자동반영 + 수동 상태변경(불량/폐기) UI** 둘 다 구현.
- 출하(delivery) 시 `consume_lots_fifo()`가 LOT을 FIFO로 소진하는 과정에서, 그 LOT에 속한
  `IN_STOCK` 시리얼 중 소진수량만큼(생성순, serial_id ASC)을 자동으로 `SHIPPED` 처리한다
  (`mark_serials_shipped=True` 플래그, 출하 흐름에서만 켬 — 생산 자재소요 등 다른 소진에는 적용 안 함).
- 수동 상태변경: `POST /api/serials/{serial_no}/status` — `IN_STOCK`/`SHIPPED`/`DEFECTIVE`/
  `SCRAPPED` 중 하나로 변경. 품질 불량 발견/폐기 결정처럼 사람이 직접 판단해야 하는 케이스를 위함.
  역할 게이팅은 QMS 등록과 동일하게 생산담당+관리자(품질담당 역할이 없어 현실적 절충).
- 재고 탭 시리얼 테이블에 상태 선택 드롭다운 + "변경" 버튼 추가(생산담당/관리자만 노출).

## 비범위
- 시리얼 상태 변경 이력(누가 언제 바꿨는지) 감사 로그 — audit_log 테이블은 있으나 이번 범위에서
  연결하지 않음
- SHIPPED 시리얼을 특정 고객/배송건과 연결하는 기능(현재는 LOT/수량 단위까지만 추적)

## 성공 기준
1. 출하 처리 시 소진 수량만큼 시리얼이 자동으로 SHIPPED, 나머지는 IN_STOCK 유지
2. 생산담당/관리자가 시리얼을 DEFECTIVE/SCRAPPED로 수동 변경 가능
3. 권한 없는 역할(영업담당 등)의 상태변경 시도는 403
4. 허용되지 않은 status 값 요청은 400
5. 재고 탭에 상태변경 UI 노출(생산담당/관리자만)
