# 03. Physical Data Model(PDM) v1.0

## 1. 목적

ERD를 실제 데이터베이스 구현 가능한 물리 모델로 전환한다.

## 2. DBMS 표준

-   Primary DB: PostgreSQL 16+
-   Cache: Redis
-   Object Storage: S3 Compatible
-   Vector DB: pgvector 또는 Milvus

## 3. Naming Convention

  항목             규칙                예시
  ---------------- ------------------- -------------
  Table            snake_case plural   materials
  PK               id                  id
  FK               {table}\_id         material_id
  Created Time     created_at          created_at
  Updated Time     updated_at          updated_at
  Logical Delete   deleted_at          deleted_at

## 4. 공통 컬럼

``` sql
id UUID PRIMARY KEY,
company_code VARCHAR(10) NOT NULL,
plant_code VARCHAR(10),
created_at TIMESTAMP NOT NULL DEFAULT NOW(),
created_by VARCHAR(50),
updated_at TIMESTAMP,
updated_by VARCHAR(50),
deleted_at TIMESTAMP,
version INT NOT NULL DEFAULT 1
```

## 5. 핵심 테이블 예시

### materials

``` sql
CREATE TABLE materials (
    id UUID PRIMARY KEY,
    material_code VARCHAR(50) NOT NULL UNIQUE,
    material_name VARCHAR(200) NOT NULL,
    material_group VARCHAR(50),
    base_uom VARCHAR(10) NOT NULL,
    material_type VARCHAR(30),
    status VARCHAR(20) NOT NULL DEFAULT 'ACTIVE',
    company_code VARCHAR(10) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP,
    deleted_at TIMESTAMP
);
```

### bom_headers

``` sql
CREATE TABLE bom_headers (
    id UUID PRIMARY KEY,
    bom_code VARCHAR(50) NOT NULL,
    material_id UUID NOT NULL REFERENCES materials(id),
    bom_type VARCHAR(20) NOT NULL,
    revision VARCHAR(20) NOT NULL,
    effective_from DATE NOT NULL,
    effective_to DATE,
    status VARCHAR(20) NOT NULL,
    company_code VARCHAR(10) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);
```

### sales_order_headers

``` sql
CREATE TABLE sales_order_headers (
    id UUID PRIMARY KEY,
    sales_order_no VARCHAR(50) NOT NULL UNIQUE,
    customer_id UUID NOT NULL,
    order_date DATE NOT NULL,
    requested_delivery_date DATE,
    currency VARCHAR(3) NOT NULL,
    total_amount NUMERIC(18,2) DEFAULT 0,
    status VARCHAR(30) NOT NULL,
    company_code VARCHAR(10) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);
```

## 6. 인덱스 전략

-   Code 컬럼 Unique Index
-   거래일자 Index
-   Company + Plant 복합 Index
-   상태값 Partial Index
-   AI Vector 검색용 ivfflat Index

## 7. Partitioning 전략

-   대용량 거래 테이블은 월별 Partition
-   inventory_transactions
-   production_results
-   accounting_document_items
-   ai_usage_logs

## 8. Audit 정책

모든 핵심 테이블은 변경 이력을 별도 history 테이블에 기록한다.
