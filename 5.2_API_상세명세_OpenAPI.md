# 02. API 상세 명세(OpenAPI) v1.0

## 1. 목적

PLM + ERP + MES + WMS + QMS + SCM + EIS + AI Agent 통합 플랫폼의 API
표준을 정의한다.

## 2. API 설계 원칙

-   REST/JSON 우선
-   OpenAPI 3.0 기반 문서화
-   OAuth2 / JWT 인증
-   API Gateway 경유
-   모든 API는 Trace ID를 포함한다.
-   대량 데이터는 Batch 또는 Event API를 사용한다.

## 3. 공통 Header

  Header           설명
  ---------------- --------------
  Authorization    Bearer Token
  X-Company-Code   법인 코드
  X-Plant-Code     공장 코드
  X-Trace-Id       추적 ID
  X-Language       언어 코드

## 5. API 예시

### Material Master API

``` yaml
openapi: 3.0.3
info:
  title: AX ERP API
  version: 1.0.0
paths:
  /api/v1/materials:
    get:
      summary: 품목 목록 조회
      parameters:
        - name: materialGroup
          in: query
          schema:
            type: string
      responses:
        "200":
          description: OK
    post:
      summary: 품목 생성
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/MaterialCreateRequest'
      responses:
        "201":
          description: Created
components:
  schemas:
    MaterialCreateRequest:
      type: object
      required:
        - materialCode
        - materialName
        - baseUom
      properties:
        materialCode:
          type: string
        materialName:
          type: string
        baseUom:
          type: string
```
